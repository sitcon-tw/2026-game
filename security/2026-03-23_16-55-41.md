# Security Review Report
- Project: 2026-game
- Stack: Go chi API + Next.js
- Context: system lifetime = 1 day
- Reviewed at: 2026-03-23_16-55-41

## Executive Summary
- 最值得先修的是「可重放的登入秘密」：player/admin/staff/booth token 直接當 session 用，且前端還把 token 放進 URL、localStorage 與 JS 可讀 cookie，當天只要截到一次就能直接接管對應身分。
- `/games/submissions` 明寫「沒有任何驗證」，可直接腳本刷等級、排名與 coupon，這對只有 1 天的活動系統仍是高價值攻擊，因為濫用會立刻影響活動公平性與折價券發放。
- 前端目前使用 `next@16.1.6`，且 `next.config.ts` 將 `/api/*` rewrite 到外部 backend；這正好落在已知 Next.js request smuggling 風險條件內，若為自架 `next start` 部署，1 天內仍值得直接升版。
- `/metrics` 與 `/docs` 在 prod 設定下對外開放，雖非直接 auth bypass，但會明顯加速攻擊者枚舉管理/工作人員流程與服務狀態。

## Findings
### [High] Token 在 URL、localStorage 與 JS 可讀 cookie 中流動，容易被重放
- Where: `frontend/app/login/page.tsx`, `frontend/app/(internal)/admin/page.tsx`, `frontend/app/(internal)/booth/page.tsx`, `frontend/app/(internal)/cashier/page.tsx`, `frontend/stores/userStore.ts`, `frontend/stores/staffStore.ts`, `frontend/stores/boothStore.ts`
- Why it matters: 活動當天只要從瀏覽器歷史、截圖、共享裝置、外掛或 XSS 拿到 token，就能直接重放成 player/staff/booth/admin 身分。
- Evidence: 多個頁面以 `searchParams.get("token")` 讀取登入 token；`userStore` 會把 token 寫進 `document.cookie` 與 Zustand persist；staff/booth token 也持久化到 localStorage。
- Fix: 停止用 query string 傳 bearer token；登入完成後立刻 `router.replace` 清掉 URL；只保留 server-set `HttpOnly; Secure` session cookie，不要把敏感 token 存進 localStorage/JS cookie。
- Confidence: High

### [High] 後端把原始 bearer secret 直接當 30 天 session 使用
- Where: `backend/internal/handler/user/login.go`, `backend/internal/handler/admin/login.go`, `backend/internal/handler/discount/staff_login.go`, `backend/internal/handler/activities/booth_login.go`, `backend/pkg/middleware/admin_key.go`, `backend/pkg/helpers/cookie.go`
- Why it matters: 一旦 token 洩漏，攻擊者可在整場活動期間持續重放；admin 更是直接把 `ADMIN_KEY` 本身變成 cookie session。
- Evidence: login handlers 直接 `SetCookie(..., token, 30*24*time.Hour)`；`AdminAuth` 以 `cookie.Value == config.Env().AdminKey` 驗證，沒有獨立 session、rotation 或 revocation。
- Fix: 改成 server-generated 短期 session ID；底層 admin/staff/booth/user secret 不直接下發到 cookie；至少縮短 TTL 並提供 logout/rotation。
- Confidence: High

### [High] `/games/submissions` 可直接刷等級、排名與 coupon
- Where: `backend/internal/handler/game/submit.go`, route `POST /api/games/submissions`
- Why it matters: 這是 1 天活動裡最容易被濫用的業務邏輯問題，會直接破壞排行榜、公平性與折價券發放。
- Evidence: 註解明寫「這整個是沒有任何驗證的」；實作只檢查 `unlock_level` 與時間差，之後就 `UpdateCurrentLevel` 並 `issueCoupons`。
- Fix: 最小修補是先停掉此 API 的 coupon issuance，或要求 server-verifiable proof/nonce；若來不及，至少加更嚴格頻率限制與異常偵測，避免腳本狂刷。
- Confidence: High

### [High] 自架 Next.js 版本命中已知 request smuggling 漏洞，且實際使用 external rewrites
- Where: `frontend/package.json`, `frontend/next.config.ts`, `frontend/Dockerfile`
- Why it matters: 若目前以前端 Node server (`next start`/standalone) 對外提供服務，攻擊者可利用 rewrite proxy 邊界差異把第二個請求 smuggle 到 backend 路由，對 `/api/*` 特別危險。
- Evidence: `next` 固定為 `16.1.6`；`next.config.ts` 把 `/api/:path*` rewrite 到 `https://2026-game.sitcon.party/api/:path*`；`pnpm audit` 回報 `GHSA-ggv3-7p47-pfv8`；`frontend/Dockerfile` 顯示為自架 standalone server。
- Fix: 直接升到 `next >= 16.1.7`；若短時間不能升，先在 edge/proxy 擋掉對 rewrite 路徑的 chunked `DELETE/OPTIONS`。
- Confidence: High

### [Medium] Prod 仍對外暴露 `/metrics` 與 `/docs`
- Where: `backend/cmd/api/main.go`, `backend/.env`
- Why it matters: 雖然不是直接入侵點，但會把路由、admin/staff 流程與服務遙測直接送給攻擊者，降低攻擊成本。
- Evidence: `r.Handle("/metrics", promhttp.Handler())` 永遠掛載；`.env` 設 `APP_ENV=prod` 且 `APP_DOCS=true`，因此 `/docs` 會公開提供 swagger。
- Fix: production 關閉 `APP_DOCS`，並把 `/metrics`/`/docs` 放到內網或管理 ACL 後面。
- Confidence: High

## Fix Priority
1. 立即要修
   - 停止 URL/localStorage/JS cookie 傳遞與保存 token
   - 把原始 bearer secret 與 session 分離，至少先縮短 TTL
   - 封住 `/games/submissions` 濫用（先停 coupon issuance 也可以）
   - 升級 `next` 到 `16.1.7+` 或在 proxy 擋掉 chunked rewrite 請求
2. 可不修（因為系統只活 1 天）
   - 細節型 security headers / CSP 強化
   - 一般性 defense-in-depth hardening，但不影響當天 exploitability 的項目

## Notes
- `backend/.env` 含真實感很高的 `ADMIN_KEY` 與 DB 連線資訊，且 `DB_SSL_MODE=disable`；`backend/.gitignore` 目前有忽略 `.env`，但仍建議人工確認這些值是否曾被 commit、打進 image 或部署到共享環境。
- `backend/fake_data/*.json` 內含可直接 import 的 user/staff/booth token；若這些檔案不是純測試資料、或曾進入正式資料庫，需一併輪替。
- 未看到明確可利用的 SQLi/SSRF/path traversal/IDOR 證據；這些點目前不建議花大量時間追。
