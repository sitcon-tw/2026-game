這個頁面列出所有可以透過打卡解鎖關卡的地點（`check` 類型活動）。你可以在這裡確認哪些地點已打卡、哪些還沒去過。

深色按鈕代表已打卡，淺色代表尚未到訪。點擊任何一個打卡點可以查看細節。

<img src="/docs/attendee/checkins.png" alt="打卡點列表頁面" width="50%" />

## 對應頁面

- `/play/checkins`

## 頁面標題

「打卡地點」

## 頁面內容

- 以兩欄格狀清單列出所有打卡點
- 已打卡：銅色底色＋白色文字，完整亮度
- 尚未到訪：淺棕色底色＋淡色文字，70% 透明度
- 點擊項目會打開詳細視窗

## 詳細視窗內容

| 欄位 | 說明 |
|------|------|
| 打卡點名稱 | 作為視窗標題 |
| 打卡狀態 | 已打卡 ✅ 或 尚未到訪 🕐 |

關閉按鈕文字：「關閉」

## 怎麼使用

1. 從 `/play` 的總覽頁確認整體進度
2. 進到這頁確認還沒去過哪些打卡點
3. 到現場找到對應地點，用 `/scan` 的掃描器掃描 QR Code 完成打卡
4. 回來確認狀態是否已更新為已打卡

## 和掃描器的關係

- 打卡動作在 `/scan` 的掃描器模式中完成（掃描現場 QR Code）
- 掃描成功後，這頁的打卡點狀態會即時更新

## 後端行為

- 資料來源：`GET /api/activities/stats`，過濾 `type === "check"`
- 打卡紀錄：`POST /api/activities/check-ins`（玩家掃描 QR Code 觸發）
- 第一次打卡：`unlock_level` 會增加，並可能取得折價券
- 重複打卡：回傳「already checked in」，不重複計算獎勵
