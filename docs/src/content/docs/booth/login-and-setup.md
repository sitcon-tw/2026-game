## 對應頁面

- `/booth?token=...`

## 功能目的

這個頁面沒有獨立登入表單，而是直接依賴 URL token。前端一進頁面就會讀 token 並登入，成功後才能開始掃碼。

## 頁面初始化流程

- 從 URL 讀取 `token`
- 把 token 存進 booth store
- 呼叫登入 API
- 成功後寫入攤位名稱與描述
- 如果失敗，就直接顯示錯誤訊息

## 成功登入後畫面會有什麼

- 攤位名稱
- 攤位描述
- 掃描器
- 訪客數字

## 失敗情境

- token 缺失
- token 錯誤
- 後端登入失敗

這些情境都不會讓使用者繼續正常操作掃碼流程。
