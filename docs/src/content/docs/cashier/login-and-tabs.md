## 對應頁面

- `/cashier?token=...`

## 登入方式

- 一進頁面就從 URL 讀取 token
- token 會被存入 staff store
- 成功登入後保留工作人員名稱與登入狀態
- 失敗時顯示登入失敗卡片

## 兩個工作分頁

- `掃描核銷`
- `發放折價券`

## 為什麼要分頁

這兩件事雖然都靠掃描完成，但掃描的 QR Code 類型不同，後續處理流程也不同，所以前端明確拆開成兩個 tab，避免現場誤用。
